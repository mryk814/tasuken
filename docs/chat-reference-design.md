# AIチャット参照棚 設計メモ

## 目的

外部AIチャットのURLを、あとで参照できる知識の束として蓄積する。

ローカルのコーディングAI作業ログは対象外とする。ローカル作業の正本はコード、コミット、ドキュメント、成果物であり、本機能ではChatGPT、Claude、Gemini、Copilotなど外部サービス上のチャットURLを扱う。

本機能の中心は「正しいフォルダへしまうこと」ではなく、「後日、なぜそのチャットを残したのかを思い出せること」である。

## 位置づけ

既存の `Theme`、`Item`、`Link` を使い、まずはスキーマ追加を最小化する。

- `Theme`: プロジェクト、研究テーマ、調査テーマ
- `Item`: テーマ内の大きな実施事項、論点、作業単位
- `Link`: 外部AIチャットURL
- `Inbox`: まだThemeやItemに振り分けていないチャットURLの入口

新しい深いディレクトリ構造は作らない。階層は `Theme -> Item -> Link` までに抑え、見つけ方は状態、要約、検索、横断ビューで補う。

## 情報設計

### チャットリンク

`Link` をAIチャット参照として扱う。既存の `link_type` に `chatgpt` があるため、最初は `chatgpt` と `other` で始められる。必要になった時点で `claude`、`gemini`、`copilot` などを追加する。

推奨フィールド:

| フィールド | 既存/追加 | 用途 |
| --- | --- | --- |
| `title` | 既存 | 後で一覧に出す短い名前 |
| `url` | 既存 | 外部チャットURL |
| `link_type` | 既存 | `chatgpt` などのサービス種別 |
| `theme_id` | 既存 | 属するテーマ |
| `item_id` | 既存 | 属する実施事項グループ |
| `description` | 既存 | 一言要約、使いどころ、判断 |
| `reference_status` | 追加候補 | `inbox`、`adopted` |
| `captured_at` | 追加候補 | 保存日。未指定時は作成日で代用 |

最初の実装では、追加候補は `data_json` 内の任意フィールドとして保存し、DB migrationを不要にする。

### 実施事項グループ

実施事項グループは新Entityではなく `Item` で表す。

推奨運用:

- `kind`: `deliverable`、`task`、`period` の既存値を使う
- `level`: `initiative` または `plan`
- `theme_id`: 所属Theme
- `description`: そのグループで何を調べる/決めるか

画面では「フォルダ」ではなく「実施事項カード」として見せる。

例:

- 仕様を固める
- 実装方針を比較する
- 実験結果の解釈を相談する
- 報告書の論点を整理する
- 採用した判断
- 捨てた案

## 画面案

### 1. チャット参照棚

テーマを選び、そのテーマ内の実施事項グループとチャットリンクを同時に見る。

```text
+--------------------------------------------------------------+
| チャット参照棚                         [チャットリンクを追加] |
| 重要な外部AIチャットをテーマと実施事項に接続します。          |
+-------------------+-----------------------+------------------+
| Theme             | 実施事項              | チャットリンク   |
|-------------------|-----------------------|------------------|
| 材料A評価         | > 仕様を固める        | [adopted]        |
| 加工条件最適化    |   3件 / 最終 6/18     | 解析条件の相談   |
| AI活用検討        |                       | ChatGPT          |
|                   | 実装方針を比較する    | 使いどころ:      |
|                   |   5件 / high 2件      | 報告書の章立て   |
|                   |                       | [開く] [コピー]  |
|                   | 捨てた案              |                  |
|                   |   1件                 | 関連リンク一覧   |
+-------------------+-----------------------+------------------+
```

ディレクトリツリーにしないため、中央列は折りたたみフォルダではなく、件数、状態、最終更新を持つカード一覧にする。

### 2. 未整理インボックス

保存の摩擦を下げる入口。URLとタイトルだけでも保存できる。

```text
+--------------------------------------------------------------+
| チャットInbox                              [URLから追加]      |
| まず残し、あとからテーマと実施事項へ整理します。              |
+--------------------------------------------------------------+
| URL / タイトル | サービス | Theme | 実施事項 | 要約 | 操作   |
|----------------|----------|-------|----------|------|--------|
| 材料A 解析相談 | ChatGPT  | 選択  | 選択     | ...  | 整理   |
| 報告書たたき台 | Claude   | 選択  | 新規作成 | ...  | 整理   |
+--------------------------------------------------------------+
```

既存 `InboxPage` は未整理ItemをLinkへ変換できるため、短期実装ではそこを拡張する。将来的にはチャット専用のInboxタブを作る。

### 3. Theme詳細への統合

Themeのホームには「最近のメモ」に加えて「最近のチャット参照」を置く。

```text
Theme: AI活用検討

現在地
...

実施事項
...

最近のチャット参照
- 仕様を固める / ChatGPT / 採用 / 6月18日
- 実装方針を比較する / Claude / 保留 / 6月17日
```

チャット参照棚は横断ビュー、Theme詳細は日常確認用とする。

## 主要操作

### 追加

1. ユーザーが外部AIチャットURLをコピーする。
2. `チャットInbox` または `チャットリンクを追加` からURLを貼る。
3. タイトル、サービス種別、一言要約を入れる。
4. Themeと実施事項は後からでもよい。
5. 保存後、正本の `Link` として残る。

### 整理

1. Inboxの未整理チャットを開く。
2. Themeを選ぶ。
3. 既存の実施事項を選ぶか、その場で新規作成する。
4. 状態を `inbox`、`adopted` から選ぶ。
5. 整理済みとしてInboxから外れる。

### 参照

1. Themeを選ぶ。
2. 実施事項カードを見る。
3. 関連チャットを開く。
4. 必要ならURLまたは一覧をクリップボードへコピーする。

### 棚卸し

1. 未整理または採用済みのチャットを横断表示する。
2. 採用済みにする、未整理へ戻す、削除する、を選ぶ。
3. 削除は論理削除とUndoを使う。

## 状態

チャットリンクの状態は、通常のタスク状態とは分ける。

| 状態 | 意味 |
| --- | --- |
| `inbox` | まだ整理していない |
| `adopted` | 何らかの判断や成果物に採用した |

色だけで区別せず、ラベルを必ず表示する。

## 検索とコピー

最初から必要な横断導線:

- タイトル、要約、URL、Theme、実施事項で検索
- サービス種別で絞り込み
- 状態で絞り込み
- 表示中のチャット一覧をTSVコピー
- 選択したチャットURLをまとめてコピー

コピー形式:

```text
タイトル	サービス	Theme	実施事項	状態	URL	要約
```

## 実装ステップ案

### Phase 1: 既存LinkのAIチャット参照化

- `Link` 型に `reference_status`、`importance`、`captured_at` を任意フィールドとして追加
- Link編集ドロワーに状態、重要度、保存日を追加
- `link_type` の選択肢に必要なら `claude`、`gemini` を追加
- Inbox整理で `link_type=chatgpt` を選びやすくする
- 保存、再読み込み、削除Undoを既存Linkとして検証

### Phase 2: チャット参照棚ページ

- 新規ルート `chat-refs`
- 左にTheme、中央に実施事項、右にチャットリンク
- Theme未設定/Item未設定のチャットを専用セクションで表示
- 一覧コピーとURLコピーを追加

### Phase 3: Inbox強化

- URL貼り付けからチャットリンクを直接作成
- 未整理チャットだけを表示するInboxフィルタ
- Theme選択、Item選択、新規Item作成を1行内で行えるようにする

### Phase 4: Theme詳細への統合

- Themeホームに最近のチャット参照を追加
- Item詳細に関連チャット参照を表示
- Relationは必要になった時点で補助的に使う

## 完成条件

- 外部AIチャットURLを保存できる
- 未整理のまま残せる
- Themeと実施事項へ後から整理できる
- チャット参照棚でTheme別、実施事項別に見られる
- 一覧コピーとURLコピーができる
- 保存後にアプリを再読み込みしても残る
- 削除後にUndoできる
- Snapshot Export/ImportでLinkの追加フィールドが保持される

## 保留事項

- サービス別のURL判定を自動化するか
- ブラウザ拡張や共有メニューから送る導線を作るか
- チャット全文の取り込みを将来扱うか
- `ai_chat` Noteとして要約本文を持つべきか、Linkの説明に留めるか
- Theme内の実施事項グループを `Item.level=initiative` に寄せるか、既存の `plan` 表示に寄せるか
